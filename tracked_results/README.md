# tracked_results

这个目录专门用于 **git 追踪的小体量正式结果同步**，目标是让两台设备都能直接拿到：

- 已经正式跑过、值得后续复盘的结果文件
- 结构化指标文件（`json` / `csv`）
- 能支持进一步分析的明细表

收录规则：

1. 只收 **正式 run**，不收临时 smoke / 草稿试跑。
2. 只收 **小体量文件**，优先 `config.json`、`summary.json`、`fold_metrics.csv`、`trials.csv` 这类可分析文本。
3. 不直接提交 `results/`、`logs/` 下的大目录，也不提交 `.log` / `.out` / `.err`。
4. 如果正式结果只有口径结论、没有现成结构化文件，允许在这里补一份精简 `metrics.json` 作为同步事实。

目录约定：

- `dl/<run_id>/`：深度学习正式 run 的配置与指标明细
- `submission/<run_id>/`：提交通道 / 全窗口 sanity 等正式结果摘要

同步流程：

1. 正式 run 跑完
2. 判断哪些文件小而值得复盘
3. 复制到本目录对应 `run_id`
4. 更新 `manifest.json`

说明：

- 这里是 **双机同步与结果管理入口**，不是替代 `docs/实验记录.md`
- `docs/实验记录.md` 负责长期文字化实验档案；这里负责随 git 同步的结果数据文件
