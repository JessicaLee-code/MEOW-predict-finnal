# MEOW 金融时序预测冲高分实验方案

## 目标

- 目标不是只把 baseline 跑通，而是把最终相关系数尽量推到 `10%+`。
- 核心策略是：`强特征 + 严格验证 + 多模型融合 + 防止泄露`。
- 先追求稳定的验证集提升，再考虑更复杂的模型。

## 先判断项目本质

当前老师给的 baseline 只有很少的特征和一个 `Ridge` 模型，属于“能跑通”的最小样例，不具备冲高分能力。

这类订单簿分钟级数据通常更适合：

- 围绕盘口不平衡、成交主动性、撤单压力、价差、波动、动量/反转做特征
- 用树模型、线性模型、轻量神经网络做对比
- 最终通过融合提升稳定性

## 我建议的整体路线

### 第一阶段：把验证体系做扎实

先不要急着堆模型，先把实验评估做对。

建议采用：

- 按时间切分训练/验证
- 留出最近一段交易日做严格验证
- 最好做 `rolling / walk-forward` 验证，而不是随机切分

原因：

- 这个任务是时间序列，随机切分很容易高估效果
- 最终比赛/提交也更看重泛化

### 第二阶段：做一套强特征体系

baseline 里只有 6 个特征，远远不够。我的建议是把特征分成 6 层。

#### 1. 盘口结构特征

围绕买卖盘深度做：

- 各档位价差
- 各档位数量差
- 各档位金额差
- 订单簿不平衡
- 深度集中度
- bid/ask 的斜率或分布形态

重点方向：

- `OBI` 类特征
- `spread` 类特征
- 深度分层差异

#### 2. 成交行为特征

围绕主动买卖、成交强弱、VWAP 等做：

- 主买/主卖数量差
- 主买/主卖成交金额差
- 主买/主卖次数差
- 成交均价与中间价的偏离
- 成交强度

这些通常能刻画短周期内资金方向。

#### 3. 撤单压力特征

撤单对短线价格影响很大，建议做：

- 买撤单/卖撤单数量差
- 买撤单/卖撤单金额差
- 撤单频率
- 撤单强度变化

这一类特征通常比单纯盘口量更有信息。

#### 4. 动量与反转特征

这是冲相关系数的重点之一。建议多尺度构造：

- 过去 1、3、5、10、20、30 分钟收益率
- 多尺度 EMA / rolling mean / rolling std
- 多尺度价格偏离度
- 短期动量和反转同时保留

原因：

- 文档本身就提示动量/反转在金融时序里很重要
- 不同股票、不同时间段可能表现不同

#### 5. 日内时序位置特征

分钟级数据有明显的日内结构，建议加：

- `interval` 的位置编码
- 开盘后第几分钟
- 离午休/收盘的距离
- 分时段分桶特征

这类特征经常被忽略，但对日内预测很有帮助。

#### 6. 横截面标准化特征

建议做同一时刻、同一天内的截面归一化：

- z-score
- rank
- percentile
- 相对行业/全市场偏离

因为老师评价的是整体预测结果，而不是单只股票局部拟合。

## 特征工程建议优先级

优先做下面这些，性价比最高：

1. 盘口不平衡 + 价差
2. 主买/主卖成交和撤单差
3. 多尺度动量/反转
4. 日内时间位置
5. 截面标准化
6. 滚动统计特征

## 模型路线

### 第一层：强基线

先训练几个稳的模型：

- `Ridge`
- `Lasso / ElasticNet`
- `LightGBM`
- `XGBoost` 或 `CatBoost`

原因：

- 线性模型便于判断特征是否有效
- 树模型通常对非线性和特征交互更敏感

### 第二层：更强的时序模型

如果特征有效，再考虑：

- MLP
- 1D Temporal CNN
- 小型 Transformer / GRU

但不建议一开始就上大模型，原因是：

- 金融分钟数据噪声大
- 小样本高维很容易过拟合
- 先有强特征再谈深模型更稳

### 第三层：融合

最终建议至少做：

- 线性模型
- 树模型
- 神经网络

三类模型的预测做加权融合或 stacking。

典型做法：

- 先用验证集确定各模型的权重
- 或者用二层线性回归做 stacking

融合的意义：

- 降低单模型不稳定性
- 对不同市场状态更鲁棒

## 我建议的实验顺序

### Experiment 0: 复现 baseline

- 跑通原始代码
- 记录训练区间和验证区间结果
- 确认指标计算一致

### Experiment 1: 特征扩充版线性模型

- 基础盘口特征
- 成交特征
- 动量/反转特征
- 截面标准化
- Ridge / ElasticNet

目标：

- 看特征是否真正有效
- 如果线性模型都能明显提升，说明特征方向是对的

### Experiment 2: LightGBM 主力模型

- 使用所有特征
- 调整叶子数、深度、学习率、采样比例
- 使用时间验证集调参

目标：

- 拿到强单模效果
- 检查是否存在局部过拟合

### Experiment 3: 加入多尺度时序特征

- 对收益率、成交差、订单簿不平衡做 rolling 特征
- 多窗口：`3 / 5 / 10 / 20 / 30`

目标：

- 捕捉短期惯性和反转
- 对不同波动状态更稳

### Experiment 4: 神经网络对照

- MLP 或小型时序网络
- 输入标准化后的特征

目标：

- 看非线性模型是否能吃到线性和树模型没吃到的模式

### Experiment 5: 融合

- `Ridge + LightGBM + MLP`
- 简单平均
- 验证集加权平均
- stacking

目标：

- 把最终相关系数推高
- 提高提交结果稳定性

## 验证和防 overfit 的原则

这题最容易犯的错是“特征很多，验证很好，最终提交崩掉”。所以必须注意：

- 不要随机打乱时间
- 不要用未来信息做归一化
- 截面统计只能用当前时刻可见数据
- rolling 特征必须严格右闭左开
- 参数调优只能看验证集，不能偷看测试集

## 我建议的最终交付结构

### 代码层

- `meow.py` 保持入口不变
- 新增特征模块
- 新增模型模块
- 新增实验配置
- 新增验证脚本

### 报告层

报告里建议明确写出：

- 数据分析结论
- 特征分组与逻辑
- 模型比较
- 融合方法
- 消融实验
- 成员分工

### 汇报层

汇报时重点讲：

- 为什么这些特征有效
- 哪些尝试有效，哪些无效
- 最终是如何把结果推上去的
- 有没有做防泄露和防 overfit 的设计

## 我对“10%+”的判断

如果只保留 baseline 风格，`10%+` 很难。

如果做到下面三件事，机会就明显上升：

1. 特征做深，尤其是盘口、成交、撤单、动量/反转、多尺度统计
2. 严格按时间验证，筛掉虚高实验
3. 最终做融合，而不是只押单一模型

我个人判断，比较稳的目标是：

- 先把验证集相关系数做到明显高于 baseline
- 然后通过融合和调参进一步冲高

## 下一步建议

如果你同意这个方向，下一步我建议我们按下面顺序推进：

1. 先把当前数据字段整理成“可做特征清单”
2. 设计训练/验证切分方案
3. 确定第一版特征工程实现
4. 跑第一个强基线
5. 再逐步加复杂模型和融合
