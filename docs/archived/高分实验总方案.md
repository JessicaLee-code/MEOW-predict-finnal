# MEOW 金融时序预测高分实验总方案

## 目标

- 在老师给定的分钟级订单簿数据上，构建一个可复现、可解释、可汇报的预测系统。
- 目标不是只超过 baseline，而是把验证集相关系数尽量推到 `10%+`，并尽可能保持测试集稳定。
- 方案设计同时服务于三件事：
  - 预测性能
  - 创新性表达
  - 报告和汇报可讲清楚

## 任务理解

老师文档里明确了几个关键点：

- 预测目标是 12 分钟后的收益率 `fret12`
- 评价看 `MSE`、`Pearson Correlation`、`R Squared`
- 方案汇报会看特征挖掘、模型定义、创新性、大模型调用、组内贡献度
- 不要 overfit，最终评估数据与提供数据不重合

因此，这个任务的正确做法不是“堆一个复杂模型”，而是：

- 先做强特征
- 再做严格时间验证
- 再做多模型融合
- 最后加一个有金融解释的创新模块

## 数据判断

从样例文件看，数据具备典型的金融微观结构特征：

- 每天大约 7 万行，约 309 到 310 只股票
- 每只股票日内大约 225 到 226 个时间点
- 字段覆盖盘口、成交、撤单、价格和多层深度
- 目标 `fret12` 分布以 0 附近为中心，长尾明显，天然噪声大

这说明：

- 单纯使用原始盘口价格不够
- 必须利用订单簿不平衡、成交主动性、撤单压力、日内位置、多尺度动量/反转
- 横截面标准化会很重要

## 总体路线

### 第一阶段：建立可靠验证

先做时间序列验证，避免随机切分。

建议切法：

- 训练期：`20230601` 到 `20231031`
- 验证期 1：`20231101` 到 `20231130`
- 验证期 2：`20231201` 到 `20231215`
- 留后测试：`20231218` 到 `20231229`

再补充 `walk-forward` 验证，用来筛特征和模型。

目的：

- 避免未来信息泄露
- 避免在单一切分上过拟合
- 让报告里的结论更可信

### 第二阶段：强特征体系

特征分成五层。

#### 1. 盘口结构特征

围绕订单簿的供需关系构造：

- 各档位买卖不平衡
- 各档位价差
- 深度集中度
- 买卖盘斜率
- 盘口厚度变化率
- spread 和 midpx 的相对位置

这层是最基础的微观结构信息。

#### 2. 成交主动性特征

围绕主买主卖构造：

- 主买/主卖数量差
- 主买/主卖金额差
- 主买/主卖次数差
- 主买/主卖高低价区间
- 成交均价相对 `midpx` 的偏离

这层通常对短周期方向判断很关键。

#### 3. 撤单压力特征

老师文档强调了市场微观结构，撤单信息是一个容易做出差异化的方向：

- 买撤单/卖撤单差
- 撤单频率
- 撤单金额强弱
- 撤单与新增单的净流入
- 买卖侧订单留存率

这部分很适合作为创新点的一部分。

#### 4. 多尺度动量与反转特征

做多窗口历史特征：

- `1 / 3 / 5 / 10 / 20 / 30` 分钟收益
- rolling mean
- rolling std
- rolling z-score
- 价格偏离均值的程度
- 动量与反转交互项

这部分要非常系统，因为老师文档明确提到了动量和反转。

#### 5. 横截面与日内位置特征

同一时刻不同股票之间的相对关系很重要：

- 当前分钟横截面 z-score
- 横截面 rank
- 横截面 percentile
- 股票相对全市场均值的偏离
- `interval` 的时间位置编码
- 开盘后、午盘、临近收盘分段特征

### 第三阶段：模型体系

建议按三层推进。

#### 1. 可解释基线

- `Ridge`
- `ElasticNet`

用途：

- 判断特征是否真的有线性信息
- 给报告提供清晰对照

#### 2. 主力模型

- `LightGBM`
- `XGBoost` 或 `CatBoost`

用途：

- 处理非线性和特征交互
- 对结构化特征通常比深度模型更稳

#### 3. 补充模型

- `MLP`
- 小型 `GRU` 或 `1D CNN`

用途：

- 捕捉树模型没有学到的模式
- 为最终融合提供互补性

## 创新设计

我建议至少保留一个“老师看得懂、也愿意认可”的创新点。

### 创新点 A：市场状态切换建模

核心思想：

- 市场在不同状态下，动量和反转的有效性不同
- 先识别状态，再针对状态做预测

状态划分可以基于：

- 波动率高低
- 盘口不平衡强弱
- 成交主动性强弱
- 撤单压力强弱

实现方式：

- 先用规则或轻量模型生成状态标签
- 再做分状态建模或 gating 融合

这类方法的优点是：

- 有金融解释
- 有创新性
- 适合写报告和讲汇报

### 创新点 B：大模型辅助因子挖掘

核心思想：

- 用大模型辅助提出因子候选
- 人工筛选保留有金融意义的特征

在报告里可以明确写：

- 大模型提出了哪些候选因子
- 哪些因子被保留
- 哪些因子被淘汰
- 为什么保留或淘汰

这正好贴合老师文档里的“大模型创新性调用”提示。

## 训练策略

建议采用以下原则：

- 所有特征只用历史可见信息
- rolling 统计必须右闭左开
- 横截面标准化只能用当前时刻可见股票集合
- 验证集调参，测试集只做最终一次提交前检查

建议损失函数和目标一致性：

- 主优化目标优先考虑 `MSE`
- 但模型筛选时要同时看 `Pearson Correlation`
- 最终模型以验证集综合表现排序

## 融合策略

最终结果建议通过融合输出，而不是押单模型。

推荐：

- `Ridge` 输出
- `LightGBM` 输出
- `MLP` 输出
- 二层 `Ridge` 或简单加权平均融合

原因：

- 单模型容易受某类市场状态影响
- 融合更稳，通常更适合最终提交

## 实验节奏

### Sprint 1

- 跑通 baseline
- 搭建严格时间验证
- 做第一版特征

### Sprint 2

- 扩充盘口、成交、撤单、多尺度动量特征
- 跑 `Ridge` 和 `LightGBM`

### Sprint 3

- 加状态切换模块
- 做消融实验

### Sprint 4

- 引入补充模型
- 做融合
- 固化最终方案

## 预期结果判断

如果只做 baseline 风格，`10%+` 基本不现实。  
如果按这份方案真正执行，尤其是：

- 特征做深
- 验证做严
- 模型做融合
- 创新点做成金融可解释的状态切换或大模型因子挖掘

那么冲到 `10%+` 是有机会的。

## 最终交付建议

代码侧保持：

- `meow.py` 作为入口
- 特征模块独立
- 模型模块独立
- 验证脚本独立

报告侧必须写：

- 数据分析
- 特征设计
- 模型对比
- 创新点
- 消融实验
- 成员分工
