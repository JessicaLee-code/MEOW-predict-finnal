# 新环境迁移说明

日期：2026-05-19  
项目：MEOW 预测研究迁移说明

## 1. 迁移目标

把当前项目拆成两部分迁移：

1. 代码与文档仓库
2. 原始数据包

代码仓库负责：

- 复现当前实验框架
- 阅读历史实验思路
- 查看阶段性结果和结论

原始数据包负责：

- 提供 `archive/` 下的 `.h5` 原始数据
- 支撑正式实验继续运行

## 2. 当前远端仓库已包含内容

当前 GitHub 仓库已推送：

- `meow-master/` 代码主干
- `run_516v3_restricted.py`
- `run_516v3_norm_only.py`
- `实验记录.md`
- `目前实验状况5.16v3.md`
- `.gitignore`

第二轮补推后，还会包含：

- 主要实验方案文档
- 阶段性汇报文档
- 小型结果 CSV
- B8 诊断结果

## 3. 新机器落地步骤

### 3.1 拉代码

```bash
git clone git@github.com:EdgarZhong/MEOW--predict.git
```

### 3.2 准备数据

当前真实数据目录是：

- `archive/`

数据规模：

- `144` 个 `.h5`
- 约 `2.45 GB`

建议做法：

1. 单独把 `archive.zip` 或整个 `archive/` 传到新机器
2. 解压或放到仓库根目录下
3. 保持结构为：

```text
MEOW--predict/
  archive/
    20230601.h5
    20230602.h5
    ...
```

### 3.3 运行代码

当前项目统一入口是：

- `python meow-master/meow-master/meow.py`

或者直接跑根目录实验脚本：

- `python run_516v3_restricted.py`
- `python run_516v3_norm_only.py`

## 5. 当前最重要的研究状态

截至当前，最可信的正式 rolling 主线仍然是：

- `R02_ridge_legacy_plus_norm_core`
- `rolling_corr_mean = 0.047976`
- `stability_score = 0.037494`

当前判断：

- OFI 暂未正式超过 R02
- `0.039211` 属于另一套 quick 评估口径，不能和正式 5 折直接并列比较
- 后续优先级仍然是：
  - R02 一致性复现
  - max_train_days 正式重测
  - OFI 去冗余
  - top legacy 精细归一化

## 6. 当前仓库的角色

当前 GitHub 仓库应该被理解成：

- 一个“代码 + 文档 + 小型结果”的研究仓库

而不是：

- 一个“包含原始 `.h5` 数据”的完整数据仓库

所以新环境要完整接手项目，必须同时具备：

1. GitHub 仓库代码
2. 本地 `archive/` 数据
