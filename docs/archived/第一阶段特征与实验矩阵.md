# 第一阶段特征与实验矩阵

## 目标

第一阶段不追求花哨，目标是快速判断哪些方向真的有效。

输出要回答三个问题：

- 哪类特征最有信号
- 哪类模型最适合当前数据
- 是否存在明显的时间泄露或过拟合

## 特征分组

### F1 盘口结构

特征内容：

- `bid0 / ask0` 价差
- 多档买卖不平衡
- 各层深度差
- 深度占比
- 盘口集中度
- spread
- midpx 相对档位位置

目的：

- 捕捉供需压力
- 为后续所有模型提供最底层信息

### F2 成交主动性

特征内容：

- `tradeBuyQty - tradeSellQty`
- `tradeBuyTurnover - tradeSellTurnover`
- 主买主卖次数差
- 主买主卖均价差
- 成交强度
- 主动成交金额占比

目的：

- 捕捉短期买卖方向
- 增强对未来 12 分钟方向的预测

### F3 撤单压力

特征内容：

- `nCxlBuy - nCxlSell`
- `cxlBuyQty - cxlSellQty`
- 撤单金额差
- 撤单频率
- 新增单和撤单的净流入

目的：

- 捕捉流动性撤退和订单簿变化
- 形成和成交特征互补的信息源

### F4 多尺度动量/反转

特征内容：

- `1 / 3 / 5 / 10 / 20 / 30` 分钟历史收益
- rolling mean
- rolling std
- rolling z-score
- 动量和反转交互项
- 价格偏离均值程度

目的：

- 利用老师文档明确提示的动量与反转逻辑
- 这是最有可能贡献增量的重要模块之一

### F5 日内与横截面

特征内容：

- `interval` 位置编码
- 开盘/午盘/尾盘分桶
- 横截面 z-score
- 横截面 rank
- 横截面 percentile
- 股票相对市场平均的残差收益

目的：

- 提供市场状态信息
- 提高泛化稳定性

## 实验矩阵

### E0 baseline 复现

- 特征：老师原始 6 个特征
- 模型：`Ridge`
- 目的：确认环境和评价一致

### E1 盘口增强

- 特征：`F1`
- 模型：`Ridge`
- 目的：验证盘口结构本身是否有增益

### E2 盘口 + 成交

- 特征：`F1 + F2`
- 模型：`Ridge`
- 目的：验证主动成交是否显著提升

### E3 盘口 + 成交 + 撤单

- 特征：`F1 + F2 + F3`
- 模型：`Ridge`
- 目的：测试微观结构信息的完整性

### E4 加入多尺度动量

- 特征：`F1 + F2 + F3 + F4`
- 模型：`Ridge`
- 目的：看动量/反转是否显著拉升相关系数

### E5 加入横截面与日内特征

- 特征：`F1 + F2 + F3 + F4 + F5`
- 模型：`Ridge`
- 目的：构建最完整的线性基线

### E6 树模型对照

- 特征：`F1 + F2 + F3 + F4 + F5`
- 模型：`LightGBM`
- 目的：验证非线性收益

### E7 非线性补充模型

- 特征：`F1 + F2 + F3 + F4 + F5`
- 模型：`MLP`
- 目的：检查神经网络是否能学到互补模式

### E8 融合

- 模型：`Ridge + LightGBM + MLP`
- 方式：加权平均或二层线性融合
- 目的：拿最终提交版本

## 必做消融

每次加一个模块都要和上一版比较。

建议最少做这些消融：

- 去掉 `F3` 看撤单是否真有价值
- 去掉 `F4` 看多尺度动量/反转是否真有价值
- 去掉 `F5` 看横截面是否真有价值
- 比较 `Ridge` 和 `LightGBM`
- 比较单模和融合

## 第一阶段输出标准

第一阶段结束时，必须得到：

- 一个严格的时间验证框架
- 一套可复用的特征生成代码
- 一个线性强基线
- 一个树模型结果
- 一份消融表

## 第一阶段决策规则

如果某类特征在多个验证切分里都不稳定，就先删掉，不要硬保留。  
如果某个模型只在单次切分上好看，也先不作为主模型。  
如果某个特征组稳定提升相关系数，就进入第二阶段继续深挖。
