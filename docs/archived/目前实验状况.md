# 当前实验状况说明

日期：2026-05-14  
项目：MEOW 金融时序预测 V3.1

## 1. 目前做到什么程度

当前这条实验线已经完成了以下工作：

- 补齐了 `V3.1` 的实验入口，可以稳定运行 `v31_quick` 和 `v31_roll`。
- 新增了几类候选方法：
  - `soft regime ensemble`
  - `common residual`
  - `pair blend`
  - rolling validation
- 加入了单日缓存，重复跑相近实验时速度明显提升。
- 把结果持续记录到 `实验记录.md`，并按 git 做了细粒度提交，便于回溯和比较。
- 评估指标不再只看单一相关系数，而是一起看 `corr / mse / r2 / daily_corr_mean / daily_corr_std`，避免只看单点结果。

## 2. 当前最好的结果

按目前的正式 holdout 切分，最好的结果来自 `soft regime ensemble`：

- `train corr = 0.3052`
- `val corr = 0.0852`
- `train r2 = 0.0932`
- `val r2 = -0.0655`
- `val mse = 0.000022`

如果只看“预测相关性做到多少”，当前大约是 `0.085`。

## 3. 当前存在的问题

现在的问题不是代码跑不通，而是结果不稳定。

- `soft regime ensemble` 在官方 holdout 上最好，但 rolling 验证波动很大。
- rolling 三折里，`soft regime` 大致是 `0.0937 / 0.0142 / 0.0169`，说明泛化不稳。
- `common residual` 在 rolling 中略更稳，但绝对提升有限，暂时还不能替代主线。
- `common residual + soft regime` 的简单融合没有带来额外收益，最优权重退化成只用 `soft regime`。
- `postprocess` 也试过了，没有稳定提升，部分设置会降低验证相关性。

## 4. 目前的判断

当前情况可以概括成一句话：

> 单次 holdout 已经到 `0.085` 左右，但 rolling 只在 `0.02~0.09` 之间波动，说明这条方案还没有形成稳定可泛化的优势。

这意味着现在不能只盯单次分数继续堆融合，而应该优先判断：

1. 这条 `V3.1` 方案是否还值得继续深挖。
2. 是否需要换成更保守、更稳定的主线。
3. 是否要回到更基础的目标定义、切分方式或特征主干上重新规划。

## 5. 建议接下来和老师讨论的重点

建议重点讨论下面这几个决策点：

- 是否继续沿着当前 `soft regime / common residual` 主线做稳定性修正。
- 是否接受当前结果已经进入平台期，转而换方案。
- 如果换方案，优先从哪一层换起：
  - 目标定义
  - 切分策略
  - 特征主干
  - 模型主干

## 6. 当前最实事求是的结论

- 这条线已经有明确正向结果，不是完全没效果。
- 但这个效果还不稳，不能当成已经“解决问题”。
- 如果目标是进一步冲更高精度，现在最重要的不是继续加复杂度，而是判断是否需要换主线。
