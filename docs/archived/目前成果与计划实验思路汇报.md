# MEOW 项目目前成果与计划实验思路汇报

更新日期：2026-05-14

## 一、项目目前已经完成的工作

### 1. 数据与实验框架已经跑通

- 已确认项目数据目录为 `archive/`，共有 `144` 个交易日的 `h5` 文件，时间范围为 `2023-06-01` 到 `2023-12-29`。
- 原始 baseline 代码已经可以直接运行，入口为 `meow.py`，原始链路为：
  - `dl.py`：读取单日/多日数据
  - `feat.py`：生成老师原始 6 个 baseline 特征
  - `mdl.py`：`Ridge` 回归
  - `eval.py`：输出 `corr / R2 / MSE`
- 当前项目不只是 baseline，可执行的增强实验框架也已经完成，核心在 `experiment_runner.py`。

### 2. 特征工程已经从 baseline 扩展到完整版本

目前代码里已经实现了较完整的分钟级特征体系，主要包括：

- `base`：盘口不平衡、成交不平衡、撤单不平衡、VWAD gap、order pressure 等基础微观结构特征
- `lag`：多窗口滞后特征
- `roll`：多窗口 rolling mean / std 特征
- `cross`：同一 `date + interval` 横截面的 z-score、rank、日内位置特征
- `regime`：波动、spread、成交活跃度、撤单压力等构成的市场状态特征

从代码结构看，当前完整特征输入已经达到约 `100+` 维：

- `base`：18 列（含元信息列）
- `base + lag`：42 列
- `base + lag + roll`：92 列
- `base + lag + roll + cross`：110 列
- `base + lag + roll + cross + regime`：121 列
- 全量版本：127 列

这说明项目已经从“老师给的最小 baseline”推进到了“可系统做消融和扩展”的阶段。

### 3. 批量实验与消融入口已经搭好

当前代码已经支持以下实验套件：

- `stage1`：baseline 到增强特征的逐步对照
- `stage2`：目标变换后的模型对照
- `ablation`：主线特征消融
- `v2`：目标分解、多尺度融合、状态融合等新版实验

支持的模型包括：

- `Ridge`
- `ElasticNet`
- `ExtraTrees`
- `GradientBoosting`
- `HistGradientBoosting`
- `LightGBM`（环境可选）
- `MLP`

### 4. V2 方案已经不是停留在想法层面，而是已有代码实现

目前 V2 相关思路已在代码中落地，包括：

- `interval_demean`：按 `date + interval` 去均值后的目标学习
- `interval_residual`：公共项 + 个股残差的目标分解
- 多尺度专家：`short / mid / long / full`
- 序列专家：`seq_short / seq_mid / seq_long`
- 状态残差融合：`run_regime_residual_fusion`
- 软门控融合：`run_soft_regime_ensemble`

也就是说，项目当前阶段已经从“做特征和单模型”进入到了“做目标分解 + 多专家融合 + 状态建模”的第二阶段。

## 二、目前已经取得的实验成果

### 1. baseline 已完成复现，并确认树模型优于线性模型

根据实验记录，在统一切分下：

- `Ridge` 验证集相关系数最高约为 `0.0839`
- `Tree (ExtraTreesRegressor)` 验证集相关系数最高约为 `0.0954`
- `ElasticNet` 不优于 `Ridge`
- `MLP` 在当前规模下明显不稳定

说明当前数据和任务下，树模型比线性模型更有潜力，`Ridge` 更适合作为稳健对照组。

### 2. 已找到当前最有效的目标变换方向

目标变换实验表明：

- 直接预测原始 `fret12` 的效果不稳定
- `interval_demean` 明显优于直接回归原始目标

在正式窗口下，当前主线结果为：

- `interval_demean + Tree + full features`
  - Train: `corr=0.2653`
  - Val: `corr=0.0882`

这说明“按分钟横截面去公共项，再学习个股相对偏离”是目前最可靠的方向。

### 3. 已完成初步特征贡献判断

从已经做过的消融看：

- `roll` 特征是当前最稳定的增益来源
- `lag` 有帮助，但边际增益不算大
- `cross` 特征在小样本冒烟测试中波动较大，但在正式窗口中仍然有一定价值

这意味着后续不能再无差别堆特征，而应保留金融解释强、在正式窗口上稳定有效的特征组。

### 4. 市场状态特征已经显示出正向价值

正式窗口下的对比结果：

- `interval_demean + Tree + no regime`
  - Val: `corr=0.0736`
- `interval_demean + Tree + regime features`
  - Val: `corr=0.0870`
- `soft regime ensemble`
  - Val: `corr=0.0891`

这个结果非常关键，说明：

- 仅仅加入状态特征就能带来正向改善
- “市场非平稳 + 分状态建模”这个方向是有效的
- 当前最值得保留为最终候选版本的，不是硬状态残差专家，而是“主树模型 + regime 辅助 + soft ensemble”

### 5. V2 复杂结构已经接入，但还没有稳定超越主线

目前已经做了：

- 目标分解
- 多尺度专家
- 序列专家
- 残差级 stacking

这些模块在冒烟测试中能够跑通，但在正式窗口上还没有稳定超过当前主线 `interval_demean + Tree + full features`。因此现阶段可以判断：

- 工程实现上已经完成了“可研究”
- 结果上还处于“有方向、未完全收敛”

## 三、我对项目当前进度和水平的判断

### 1. 当前项目完成度估计

如果把整个任务拆成“能跑通 baseline -> 建立增强特征 -> 找到稳定主线 -> 做创新实验 -> 收敛最终提交版本”五个阶段，我认为当前完成度大约在：

`70% - 75%`

理由是：

- baseline 已复现
- 特征工程和批量实验器已完成
- 主线模型已经明确
- 创新方向已经开始落地
- 但最终版尚未在更严格窗口下完成稳定收敛

### 2. 当前预测效果达到了什么水平

如果以“验证集相关系数是否接近 10%”作为阶段性目标，那么目前最可靠成绩约为：

- 当前较优结果：`Val corr = 0.0891`
- 与 `10%` 目标的差距约为：`0.0109`

因此可以比较保守地说：

- 目前已经基本逼近 `10%` 目标
- 但还不能说已经稳定达到 `10%+`
- 更准确的说法应是：已经找到接近目标的可行主线，下一步重点是把结果做稳，而不是盲目继续加复杂度

### 3. 当前最重要的阶段性成果

我认为目前最有价值的成果不是某一个单点分数，而是以下三点：

- 已经证明 `interval_demean + Tree + full/regime features` 是真实有效主线
- 已经证明“状态特征/soft gating”是目前最有希望继续抬升结果的创新方向
- 已经把多尺度、序列、目标分解等后续方向全部接入统一实验框架，后面可以快速迭代

## 四、当前存在的问题

- 当前结果仍然对时间切分比较敏感，说明问题本质上仍然是非平稳和泛化不足
- V2 复杂结构在冒烟测试中训练集提升明显，但验证集不稳定，存在过拟合风险
- 多尺度和序列专家虽然已经接入，但目前还没有在正式窗口里证明自己能稳定带来独立增益
- `R2` 目前普遍偏弱，说明模型虽然在排序/相关性上有改善，但对数值拟合的稳定性还不够强

## 五、最新的实验实施思路

### 1. 主线不再改动为“大换模型”，而是做稳健增强

接下来不建议再把重点放在“换一个更复杂的深度模型”上，而建议围绕当前最有效主线继续优化：

- 主线模型：`interval_demean + Tree + full features`
- 强化方向：加入 `regime features`
- 当前候选提交版：`main tree + regime soft ensemble`

原因是这条线已经在正式窗口下给出了最可信的正向结果。

### 2. 最新实验重点应放在 soft regime ensemble 的稳健化

具体建议如下：

1. 固定正式时间切分，优先使用 `8` 天训练、`2` 天验证的正式窗口做判断。
2. 把 `main tree` 和 `regime tree` 作为两个稳定专家，不再增加过多新专家。
3. 对 soft ensemble 做更稳健的二层融合：
   - 优先简单 `Ridge` 融合
   - 再尝试轻量树融合
   - 重点看验证集相关系数是否稳定超过 `0.0891`
4. 对 regime 定义做小步微调，而不是重写整个模型：
   - 波动率分层
   - spread 分层
   - order pressure 分层
   - 日内时段分层

### 3. V2 模块的推进方式应改为“辅助冲分”，而不是“替代主线”

对以下模块的建议是：

- `interval_residual`：继续保留，因为它理论合理、代码已接好，但必须用正式窗口验证
- 多尺度专家：优先保留，因为与现有特征体系最契合
- 序列专家：暂时作为辅助分支，不作为主模型核心
- 状态残差专家：目前不建议作为最终汇报主方案

也就是说，接下来的策略应是：

- 先稳住主线
- 再让 V2 模块逐个尝试提供“额外增益”
- 任何模块若不能在正式窗口稳定提升，就只写为尝试，不进入最终提交版本

### 4. 下一阶段最推荐的具体实验顺序

建议按下面顺序继续：

1. 固化正式评价窗口，统一使用同一套切分做对照
2. 复跑 `main tree`、`regime tree`、`soft regime ensemble`
3. 对 regime 特征做小范围调参和定义微调
4. 在主线稳定后，再测试 `interval_residual + multiscale fusion`
5. 只有当某个 V2 模块在正式窗口上稳定超过主线时，才升级为主候选

## 六、现阶段适合和老师讨论的问题

我建议和老师重点讨论以下几个问题：

### 1. 最终评价优先级

是继续以 `corr` 作为第一指标，还是要同时更强调 `R2 / MSE`。  
因为当前很多实验是“相关性提升明显，但 R2 不同步改善”。

### 2. 最终汇报更偏“稳健主线”还是“创新结构”

目前项目有两种表达方式：

- 稳健路线：突出 `interval_demean + Tree + regime + soft ensemble`
- 创新路线：突出目标分解、多尺度专家、状态门控

如果老师更重视结果稳定性，建议主报告以稳健路线为主；  
如果老师更重视研究表达和创新性，可以把 V2 作为第二部分重点展示。

### 3. 是否接受“先逼近 10%，再做最后冲刺”

从现有结果看，项目已经非常接近 `10%`。  
我认为下一步最合理的目标不是同时全面铺开，而是：

- 先把 `0.0891` 这条线做稳
- 再用少量高价值实验去尝试冲到 `10%+`

## 七、简要结论

目前这个项目已经不处于“刚起步”阶段，而是已经完成了 baseline 复现、增强特征体系、统一实验框架和 V2 创新入口的搭建。当前最可信的主线已经明确为：

`interval_demean + Tree + full/regime features + soft ensemble`

按当前进度判断，项目完成度大约在 `70% - 75%`，验证集最优结果已经逼近 `10%` 目标。后续最重要的任务不是继续盲目堆复杂模型，而是把现有有效主线做稳，并验证哪些 V2 模块能带来真实、可复现的增益。
