# 目前实验状况 5.16 V1 报告

日期：2026-05-16  
项目：MEOW 金融时序预测 V3.3  
用途：给老师讨论当前正式 rolling 结果、风险判断与下一步实验路线

## 1. 当前阶段判断

当前阶段已经不再适合继续盲目扩展复杂模型。Stage 0 quick rolling 和后续正式 rolling 都说明：

- 复杂模块没有稳定超过最朴素的 `B0_ridge_legacy`
- 当前可复现的稳定相关系数仍在 `0.04` 到 `0.05` 附近
- 距离 `0.10` 仍然有明显差距

因此，当前目标不是“马上冲更高分”，而是：

1. 找到正式 rolling 下最稳定的 backbone
2. 解释 legacy 特征为什么比 core 更强
3. 只做受限的特征增强和受限融合

## 2. 正式 rolling 复验设置

本轮实验统一采用：

- 5 折 formal rolling
- `train_window=8`
- `val_window=2`
- `step=10`
- `embargo=1`
- `max_train_days=4`
- 统一切分、统一随机种子、统一特征处理

评价指标：

```text
stability_score = rolling_corr_mean - 0.7 * rolling_corr_std
```

同时记录：

- `rolling_corr_mean`
- `rolling_corr_std`
- `rolling_corr_min`
- `rolling_mse_mean`
- `rolling_r2_mean`
- `daily_corr_mean`
- `daily_corr_std`

- `B8_ridge_legacy_plus_core` 没有稳定超过 `B0`，而且波动更大

## 5. 为什么 legacy 比 core 强

### 5.1 legacy 中而 core 中没有的特征

- `ob_imb0`
- `ob_imb4`
- `ob_imb9`
- `trade_imbema5`
- `lagret12`

其中 `trade_imb` 同时出现在 legacy 和 core 中。

### 5.2 信号含义

- `ob_imb0` / `ob_imb4` / `ob_imb9`：不同深度的订单簿 imbalance
- `trade_imbema5`：成交 imbalance 的平滑版本
- `lagret12`：12 分钟收益偏离，反映中短周期路径信息

### 5.3 Ridge 系数摘要

| feature | mean_abs_coef | mean_abs_rank | sign_consistency | fold_stability |
| --- | ---: | ---: | ---: | ---: |
| `trade_turnover_imb` | 0.0006136 | 7.20 | 0.8 | 0.99940 |
| `trade_imb` | 0.0005949 | 2.60 | 0.8 | 0.99973 |
| `ob_imb4` | 0.0005549 | 4.80 | 0.6 | 0.99946 |
| `obi4` | 0.0005539 | 4.60 | 0.6 | 0.99946 |
| `obi0` | 0.0004580 | 7.20 | 0.8 | 0.99940 |
| `ob_imb0` | 0.0004469 | 8.20 | 0.8 | 0.99942 |
| `order_pressure` | 0.0003407 | 7.20 | 0.8 | 0.99971 |
| `trade_imbema5` | 0.0002740 | 6.00 | 0.4 | 0.99986 |
| `lagret12` | 0.0001404 | 11.80 | 0.6 | 0.99984 |

### 5.4 初步解释

legacy 更强的原因不是“更复杂”，而是更贴近：

- 订单簿 imbalance
- 成交 imbalance
- imbalance 的平滑形式

这说明当前有效信号更偏向“供需压力”和“成交方向性”，而不是更复杂的非线性结构。

## 7. 训练窗敏感性结果

为了验证 `max_train_days=4` 是否压制性能，本轮做了最小正式对照：

- `max_train_days=4`
- `max_train_days=80`

### 7.1 结果

| experiment | rolling_corr_mean | rolling_corr_std | rolling_corr_min | stability_score |
| --- | ---: | ---: | ---: | ---: |
| `R02_ridge_legacy_plus_norm_core_max_train_4` | 0.039211 | 0.002293 | 0.036918 | 0.037606 |
| `R02_ridge_legacy_plus_norm_core_max_train_80` | 0.026005 | 0.016894 | 0.009110 | 0.014179 |

### 7.2 结论

- 在这次对照里，`max_train_days=4` 反而比 `80` 更稳
- 所以当前不能简单认为“训练窗越大越好”
- 更长训练窗可能引入过多旧分布样本，稀释了当前有效信号

## 8. 当前阶段结论

当前最重要的结论是：

1. `B0_ridge_legacy` 仍然是最稳 backbone
2. `R02_ridge_legacy_plus_norm_core` 是当前最强 Ridge 增强方向
3. OFI 有增量，但还不足以替代 backbone
4. `max_train_days=4` 并没有表现出明显压制，至少在这次对照中比 `80` 更好

## 9. 下一步建议

接下来建议只做三件事：

1. 继续围绕 `B0` 和 `R02` 做受限增强，不再盲目扩复杂模型
2. 深挖 `legacy` 和 `normalized core` 为什么有效，判断哪些是稳定信号，哪些只是噪声
3. 如果要继续冲更高分，优先检查特征对齐、切分、泄漏和信号编码，而不是直接加更复杂的模型结构

## 10. 可直接对老师汇报的一句话

> 当前正式 rolling 下最稳的 backbone 仍然是 `B0_ridge_legacy`，最好的 Ridge 增强是 `legacy + normalized core`，OFI 有一点增量但没有超过当前最佳，训练窗方面 `max_train_days=4` 反而比 `80` 更稳。现在更关键的是继续找稳定信号源，而不是继续堆复杂模型。
