# MEOW 金融时序预测高分实验总方案 V2

## 目标

- 以预测效果为第一优先级，目标是在验证集上稳定提升相关系数，并尽量逼近或超过 10%。
- 方案同时兼顾可解释性、论文方法融合度和最终汇报时的创新性表达。
- 不追求单一大模型，而是采用“强特征 + 多尺度序列模型 + 状态切换 + 融合”的组合路线。

## 总体判断

- 这类分钟级股票预测问题的核心难点不是模型表达力不够，而是非平稳、噪声大、市场状态切换快。
- 单靠树模型容易吃到局部静态特征，但对状态切换和序列结构利用不够。
- 单靠深度模型容易过拟合，尤其在样本窗口有限、未来分布漂移明显时。
- 最优路线应当是把两类模型结合起来，再用稳健的目标分解和分状态机制降低漂移影响。

## 论文精华映射

### DeepLOB / TransLOB

- 借鉴点：直接建模 limit order book 的空间结构与短时序依赖。
- 落地方向：用历史窗口构造 `time x feature` 输入，采用 `causal conv + attention` 或轻量 1D CNN + attention。
- 适合做一个序列分支，补足现有 tabular 特征模型对局部时序结构的不足。

### PatchTST / LiT

- 借鉴点：把连续时间序列切成 patch，减少 token 数并保留局部语义。
- 落地方向：把过去 30/60 分钟特征按时间 patch 化后输入 Transformer。
- 适合做一个轻量 patch 分支，不建议硬追求重型 Transformer。

### TimesNet / TimeMixer

- 借鉴点：多尺度、分解、不同采样粒度下的信息互补。
- 落地方向：把 15/30/60 分钟窗口、lag 特征、roll 特征分别作为不同尺度来源，最后融合。
- 这条线和你现有的多尺度特征最匹配，工程风险最低。

### FOIL / OOD Generalization

- 借鉴点：把时间序列中的不同阶段视作不同 environment，学更稳健的 invariant feature。
- 落地方向：按月份、波动率、盘口不平衡、撤单压力、日内时段构造 regime，再做 gating 或分专家建模。
- 这部分对金融非平稳问题非常合适，也是报告里最容易讲清楚的创新点。

## V2 主线结构

### 主线 A：目标分解

- 第一步预测公共项，也就是 `date + interval` 层面的市场共同成分。
- 第二步预测个体残差，也就是单只股票相对公共项的偏离。
- 最终输出等于公共项预测加上残差预测。
- 这比直接训练 `interval_demean` 更完整，也更接近“分解 + 重建”的论文思路。

### 主线 B：多专家建模

- 建立一个强 tabular 专家，继续使用你现在的特征体系和树模型。
- 建立一个序列专家，吃原始分钟窗口或 patch 化窗口。
- 建立一个多尺度专家，分别看短窗、中窗、长窗。
- 用 gating 或二层融合把多个专家输出合并。

### 主线 C：状态切换

- 把样本按市场状态切开，而不是所有样本共用一套参数。
- 状态建议先从规则做起，避免一开始过于复杂。
- 可用状态特征包括：
- 波动率高低
- spread 大小
- 盘口不平衡强弱
- 成交主动性强弱
- 撤单压力强弱
- 日内位置

## 具体模型设计

### 模型 1：Tabular 强基线

- 保留当前 `Tree + interval_demean + full features`。
- 继续尝试 `ExtraTrees`, `LightGBM`, `CatBoost` 中能稳定跑通的那个。
- 重点做参数稳健化，而不是一味加深树。
- 这个模型负责吃手工特征，作为融合中的稳定锚点。

### 模型 2：LOB 序列模型

- 输入过去若干分钟的原始或半原始 LOB 信息。
- 推荐优先使用较轻的结构：
- `causal conv + attention`
- 或 `1D CNN + attention`
- 或 `patch embedding + Transformer`
- 不建议一开始就上深层 Transformer。
- 序列长度优先从 30、60 分钟试起，过长容易噪声更大。

### 模型 3：多尺度模型

- 为每个样本构造多个时间尺度视角。
- 例如短窗捕捉冲击，中窗捕捉惯性，长窗捕捉反转。
- 每个尺度各自训练一个子模型，最后做融合。
- 这条线最接近 TimeMixer / TimesNet 的思想，且容易做消融。

### 模型 4：状态门控模型

- 用一个轻量分类器或规则打分器判断当前样本属于哪类市场状态。
- 每个状态对应不同专家，或者给不同专家分配不同权重。
- 这相当于把非平稳问题显式分解掉。

## 特征方案

### 保留特征

- 盘口不平衡
- 成交不平衡
- 撤单不平衡
- 买卖 VWAD gap
- `lag` 特征
- `roll` 特征
- 横截面特征
- 日内位置特征

### 新增特征

- 多窗口收益率和波动率
- 多窗口 order flow 统计
- 多窗口盘口斜率或厚度变化率
- 市场状态标签相关特征
- 分环境标准化特征

### 不建议继续堆的特征

- 低信噪比的交叉项
- 纯随机组合出来但没有金融解释的高阶项
- 重复表达同一信息的多份相似特征

## 实验顺序

### 阶段 1：稳住当前主线

- 目标是确认现有方案在更稳定切分下仍然有效。
- 跑通 `Tree + interval_demean + full features`。
- 跑通 `Ridge` 与 `Tree` 的统一对照。
- 固化一份正式的时间切分方案。

### 阶段 2：做目标分解

- 先做 `common component + residual` 的双阶段方案。
- 比较下面三种：
- 直接回归原始 `fret12`
- 只回归 `interval_demean`
- `common + residual` 双阶段回归

### 阶段 3：上序列专家

- 先做轻量版 `LOB-lite`。
- 输入用历史窗口，不要直接全日堆入。
- 和 tabular 专家做并行训练，再看融合收益。

### 阶段 4：上多尺度专家

- 构造 15/30/60 分钟多窗口特征或多窗口分支。
- 先做简单平均融合，再做二层融合。

### 阶段 5：加入状态门控

- 用规则状态作为第一版 gating。
- 如果有效，再考虑学习式 gating。
- 这一步是冲高分创新点的核心。

### 阶段 6：最终融合

- 融合 tabular 专家、序列专家、多尺度专家、状态专家。
- 优先尝试简单线性融合或二层 Ridge。
- 不要一开始就用复杂 stacking 网络。

## 实验矩阵

### 基线组

- `B0`: 原始 6 特征 + Ridge
- `B1`: full features + Ridge
- `B2`: full features + Tree
- `B3`: full features + interval_demean + Tree

### 分解组

- `D1`: 只回归 `interval_demean`
- `D2`: `common component + residual`
- `D3`: `date_demean`

### 序列组

- `S1`: `causal conv + attention`
- `S2`: `patch Transformer`
- `S3`: `DeepLOB-lite`

### 多尺度组

- `M1`: 单尺度
- `M2`: 双尺度
- `M3`: 三尺度
- `M4`: 多尺度融合

### 状态组

- `R1`: 规则状态分组
- `R2`: 规则状态 + 分专家
- `R3`: 学习式 gating

### 融合组

- `F1`: Tree + 序列模型
- `F2`: Tree + 多尺度模型
- `F3`: Tree + 序列 + 多尺度
- `F4`: 全量融合

## 消融原则

- 每次只新增一个模块，避免无法判断增益来源。
- 如果一个模块只在单次切分上有提升，但跨切分不稳定，就不保留为主线。
- 如果某个模块和 `interval_demean` 高度耦合，就单独评估它是否真的贡献独立增益。
- 如果某个深度模型训练不稳，优先减小结构而不是强行堆层数。

## 推荐优先级

1. 目标分解 + 残差建模
2. 多尺度专家
3. 规则状态 gating
4. 轻量序列模型
5. 最终融合

## 预期输出

- 一条稳定的主线预测方案。
- 一张清晰的消融表，能说明各模块的独立贡献。
- 一套可以在汇报里直接讲的创新路径。
- 一个最终提交版本，优先保证泛化而不是训练集分数。

## 结论

- V2 的核心不是“再造一个复杂模型”，而是把顶尖论文中真正对这题有用的部分拆出来重组。
- 最有价值的迁移方向是：
- LOB 序列结构建模
- 多尺度建模
- 分环境稳健学习
- 目标分解与融合
- 这些方法叠加后，比单纯堆特征或堆深度模型更有机会提升最终预测表现。
