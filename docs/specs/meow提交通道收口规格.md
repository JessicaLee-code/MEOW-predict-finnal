# meow 提交通道收口规格

更新日期：2026-05-25

## 1. 目标

把 `meow/` 收成老师最终可直接运行的正式提交入口包装层，同时保持 `src/` 为唯一核心管线实现，消灭“实验 driver 一套、提交 driver 一套”的双实现风险。

老师最终口令已明确为：

```bash
python meow.py
```

因此本规格的核心要求不是更换入口，而是：

- 保持 `meow.py` 入口、调用顺序、文件命名风格兼容老师样例
- 让 `meow.py` 最终实际运行到我们自己的正式核心实现

## 2. 设计边界

### 2.1 哪些必须共用

以下内容属于核心管线，必须只保留一套实现，并放在 `src/` 内：

- 特征构造公式
- 训练标签处理（含 winsorize）
- 模型训练
- 推理前输入列处理
- 模型预测
- 预测后处理
- 最终 `forecast` 输出口径

### 2.2 哪些允许分叉

只允许在下面两处做 driver 级分叉：

1. **任务编排**
   - 实验 driver：rolling / holdout / leaderboard / 并行调度
   - 提交 driver：单次 `fit` / 单次 `eval`

2. **特征物化方式**
   - 实验 driver：允许优先读硬盘特征缓存
   - 提交 driver：必须原始数据现算特征，不能依赖持久化特征缓存

注意：这里的“分叉”是模式选择，不是两套独立算法实现。

## 3. 仓库归并要求

### 3.1 `meow/` 纳入主仓库管理

当前 `meow/` 是游离目录/嵌套仓库状态，主仓库未正式跟踪。需要完成：

- 清理 `meow/.git` 的嵌套仓库状态
- 让 `meow/` 文件由当前主仓库统一跟踪
- 后续所有提交链改动都在主仓库版本历史中可见

### 3.2 不新增第二套“提交专用仓库”

禁止再创建新的 `submission/` 或其他并行入口目录。老师最终入口就是 `meow/`。

## 4. 包装层改造要求

### 4.1 `meow.py`

保留老师样例风格：

- `MeowEngine`
- `fit(startDate, endDate)`
- `predict(xdf)`
- `eval(startDate, endDate)`

允许改内部实现，但不改外部使用方式。

### 4.2 `meow/feat.py`

角色改为“提交特征包装层”：

- 保留 `MeowFeatureGenerator` / `genFeatures`
- 内部委托到 `src/` 的正式特征实现
- 不能再维护独立 baseline 特征公式

### 4.3 `meow/mdl.py`

角色改为“提交训练/推理包装层”：

- 保留 `MeowModel`
- `fit()` 内部调用 `src/` 的正式训练实现
- `predict()` 内部调用 `src/` 的正式推理实现
- 训练阶段必须使用与实验链一致的 winsorize 口径

### 4.4 `meow/eval.py`

外部行为保持兼容即可。只要老师最终评测方式仍是 `forecast` vs `fret12`，可保留其壳层职责。

## 5. 核心实现要求

### 5.1 `src/` 作为唯一真后端

需要在 `src/` 中补齐一层很薄的提交适配能力，但禁止大规模重构现有实验主链。

原则：

- 不为“架构漂亮”重写 `experiment_runner` / `eval_protocol` / `scheduler`
- 以最小桥接代码复用现有可用核心
- 新增的桥接层要服务于 `meow/`，不是再造第三套系统

### 5.2 提交链禁止依赖持久化特征缓存

老师会换数据路径后直接运行 `python meow.py`。因此：

- 不能依赖 `data/features/`
- 不能依赖本地预构建 stage artifact
- 不能要求提交包里包含大体积特征缓存

允许的只有：

- 从老师给的新原始 `.h5` 数据直接现算特征
- 可选做进程内/临时内存级缓存

不允许做：

- 假设磁盘上已经有旧特征缓存
- 用本地历史缓存直接服务老师新数据

## 6. winsorize 口径要求

训练标签 winsorize 已在实验链落地。收口后必须满足：

- `meow.py` 的训练阶段也真实使用同一套 winsorize 逻辑
- 默认配置与实验链一致
- 参数来源一致，不能一边开一边关
- 只作用于训练标签，不改最终输出尺度

## 7. 验收标准

### 7.1 结构验收

- `meow/` 已被主仓库跟踪
- `meow/` 不再是游离嵌套仓库
- 提交链核心逻辑不再维护独立 baseline 实现

### 7.2 行为验收

在真实数据上执行：

```bash
cd meow
python meow.py
```

要求：

- 能直接训练
- 能直接预测
- 能直接评测
- `forecast` 行序与 `xdf` 严格对齐
- 每行输出有限值

### 7.3 一致性验收

在同一数据切分下，至少核对以下一致性：

- 特征列集合一致
- winsorize 开关与分位一致
- 训练目标口径一致
- 预测输出口径一致

### 7.4 缓存验收

- 删除或不提供 `data/features/` 时，`meow.py` 仍可工作
- 不依赖任何提交包外的本地缓存/隐藏状态

## 8. 工作量预估

- 仓库归并：0.5 小时
- 包装层桥接实现：2~4 小时
- `meow.py` 真实数据闭环与修 bug：1~3 小时

总计：

- 顺利时：半天
- 若出现 train/serve skew、缓存耦合、列对齐问题：按 1 天预估
