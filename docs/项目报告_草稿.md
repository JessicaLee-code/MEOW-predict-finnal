## 1. 方法介绍（总览）

数据为约 300 只股票、2023-06 至 2023-12 的分钟级盘口数据，预测目标为 12 分钟前向收益率。最终交付方案为一套
**「传统多因子保底 + 深度学习去相关增强」的等权融合**：

- **传统支路（保底，可解释，无条件可用）**：在 433 维人工特征上做**两成员等权融合** —— `X1`（`StandardScaler → Ridge`
  线性多因子）与 `M_lgbm_d4`（深度 4 的 LightGBM 梯度提升树）；融合口径为 `raw_mean`（等权原值平均，保留 `fret12`
  量纲以同时维护 MSE 与 R²）。该支路跨折稳定为正，构成交付下限。
- **深度学习支路（增强，去相关，依赖 PyTorch 与 GPU）**：一个截面序列模型，**直接以原始盘口 raw 通道（59 维微结构通道）为输入**，
  而非 433 维人工摘要。其预测与传统支路**中等去相关（ρ≈0.45–0.55）**，融合带来净增益而非冗余。
- **融合规则**：`forecast = 0.5·传统 + 0.5·深度学习`（等权、零自由参数）；深度学习因序列 warmup 而缺失的行由纯传统预测填充，不产生 NaN。

工程上，深度学习支路以**防御式**方式接入正式入口 `meow.py`：当 torch、CUDA 或任一深度学习环节出错时自动回落至纯传统预测
（最坏情形即传统保底分），保证 `python meow.py` 在任意 Python≥3.8 环境下稳定运行、不产生 NaN。

代码骨架（老师可按文件名与类名定位）：入口 `meow/meow.py` 的 `MeowEngine`（`fit`/`predict`/`eval`，签名与初始代码一致）；
传统链 `src/submission_pipeline.py`、`meow/mdl.py` 的 `MeowModel`、`meow/feat.py` 的 `MeowFeatureGenerator`、`src/experiment_runner.py`；
深度学习支路 `meow/dl_serve.py` 的 `DLServe` 与 `fuse_traditional_with_dl`；深度学习卡带 `models/dl_models.py`（`CrossSectionCartridge` / `RawChannelAdapter`）及序列地基 `src/sequence_dataset.py`。

## 2. 数据分析（EDA 与初步认知）

- **数据本体为富盘口数据**：多档 LOB 价量（bid0–4 / ask0–4 及各档数量、金额）、挂撤单订单流（add / cxl / trade，区分主买主卖）
  与 226 步日内序列，约 59 个原始微结构通道。项目早期一度误判为「无 LOB / 数据锁死」，经系统盘点后予以纠正：
  **raw 信息量远超 433 人工特征的利用程度**（挂撤单 20 列仅被压缩为 4 个静态比率、盘口形状未建模、完整序列被压成统计量）——
  详见 `docs/原始数据盘点与盘口建模诊断.md`。这一认知直接驱动了后期「不在人工摘要上精细优化、转而直接建模 raw」的战略调整。
- **基准认知**：老师给出的参考为——原始 6 特征 + Ridge 仅 2%–3% corr；业内顶级水平 20%–30%；对本项目的预期为 5%–10%，超过 10% 加分。
  同数据同题目存在 12%+ 的实现，说明本项目与之的差距源于**方法不足，而非数据上限**。
- **标签性质**：`fret12` 为 12 分钟前向收益，日内不跨夜；分布长尾、序列非平稳、信噪比极低——
  因此目标为**追求稳健的相关性而非高准确率**，并须严格防范时段过拟合。
- **关键 EDA 发现（早期实验，5/11）**：对训练目标做 `interval_demean`（在每个 `date+interval` 横截面内去均值）后，
  验证集相关性显著优于直接回归原始 `fret12`。**这说明真正可学的信号是「同一时刻、不同个股之间的相对强弱（截面 alpha）」，
  而非绝对收益值**——该发现奠定了后续全部工作的主轴（传统侧以 cross-z / rank 显式建模截面，深度学习侧以 set-attention 截面支路建模）。

## 3. 特征工程

### 3.1 433 维人工特征（9 个 stage，`src/feature_registry.py` / `FeatureStore` / `FeatureLoader`）

每个特征族均有明确的市场微观结构依据，而非随机组合：

| Stage | 逻辑来源 | 代表特征 |
|---|---|---|
| `base`/`legacy` | 原始价量基础 + 12 步滞后收益（与预测窗同尺度的动量/反转主体） | `lagret12`、midpx 衍生 |
| `lag` / `roll` | 多窗口滞后/滚动统计——动量与反转效应的多尺度刻画 | `lagret{1,3,6,12,24}`、滚动波动 |
| `ofi` | 订单流不平衡（Order Flow Imbalance）——买卖压力净流向 | `ofi_*`、`ofi_*_cs_z` |
| `trade_impact` | 成交冲击——主买主卖强度、成交压力 | `trade_pressure_*`、`trade_imb` |
| `cross` | **截面归一化 / 排名**（cross-z / cross-rank）——同一时刻个股间相对强弱 | `*_cs_z`、`*_cs_rank` |
| `conditional_momentum` | **条件动量**——动量×微结构状态交互（动量仅在特定状态下有效） | `lagret×{ofi,spread,trade_pressure,vol}` |
| `regime` | 市场状态/制度特征 | `reversal_state`、`state_pressure` |
| `patch` | 日内分段摘要 | 各 patch last/summary |

### 3.2 系统化特征验收（P1–Q4，"宽进严出"）

特征不依主观臆断采纳，而须通过一套可复现的评测漏斗（`experiments/p0_eval_protocol.py` + baseline delta + 每日 IC-IR + expanding 关口）：

- **P1 OFI 族（O1–O6 六变体）** 与 **P2 成交冲击族（T1–T4）**：单族追加在线性 Ridge 下基本判负。
  **根因一致**——每族最可外推的主体（截面归一化版）已包含于基线 `norm_core`；再追加原始 / 动态 / depth 变体或为零增益，
  或损害最坏折（P2 的 T1 制造出新的强负折，即「表面提升、实则损害最坏折」）。
- **P3 条件动量**：C1 全集因数据需求过高而过拟合判负；**C2 交互项（`lagret×{ofi,spread,trade_pressure,vol}`）增益虽小但稳健**
  ——跨 profile 符号一致、每日 IC 双升、最坏折改善，是 P1 以来最稳健的候选。
- **Q2 alpha 敏感性**：扫描 alpha=2/5/20，确认 alpha 非关键杠杆、弱信号并非欠正则所致。
- **Q3 P3.5 高先验交互（新建 4 项）**：线性下零增益——源特征基线已含 cross-z 线性项，乘积无独立信号。
- **Q4 跨族组合 X1 = R02 + ofi_safe + 条件动量交互**：daily 首次突破地板（Δ+0.0031），**expanding 关口 Δ+0.0044、
  12 折全正、配对 t=7.35、过拟合极小**——为 P1–Q4 中唯一通过 expanding 的候选，且增益随训练历史延长单调增大（最契合泛化目标）。
- **树重要性扫描（Fork A 反偏颇诊断，`experiments/p4_tree_importance.py`）**：一次性回答「树模型是否用得上人工交互、线性是否错杀 ofi / trade」
  ——结论为：人工交互对树模型冗余（占比 0.9%）、成交冲击被线性低估（占比 24.9%，第二名）、raw OFI 全形态对树模型贡献均排名最末。

### 3.3 rx_micro 新特征族（补充被低估的 raw 信号）

针对 §2 发现的「433 严重低估 raw」，补充一组 49 列特征（`rx_` / `rx2_` / `rx3_`：归一化 / 比率 / 不平衡 / 相对量 / OHLC 及分档成交额、挂撤单价格区间）。
三折全部为正（lgbm 0.0765→0.0799，新特征重要性占比稳定为 13.4%，属真实信号），接入交付链后 Dec 全窗 sanity 由 0.0803 提升至 0.0812。
但**静态摘要的增量空间有限（+0.0034）**，反向印证 raw 缺失价值的主体在于时序动态——这正是深度学习直接建模 raw 的动机。

## 4. 模型定义、选择与创新设计

### 4.1 探索历程（含有价值的失败，供"探索度"评估）

模型选择经历多轮诚实试错，每一步均由上一步的失败根因驱动。**以下负结果均如实保留，作为方法广度与探索深度的证据。**

**A. 传统 / 集成阶段（5/11–5/29）**

1. **模型横向对比**（`Ridge` / `ElasticNet` / `ExtraTrees` / `MLP` / `HistGradientBoosting`）：树模型系最佳，MLP 在当前特征与规模下性价比不足，ElasticNet 与 Ridge 相当。
2. **目标变换**：`interval_demean`（截面去均值）显著优于直接回归；亦尝试 `interval_residual`（残差目标分解）。
3. **Regime 路线**：regime 特征、regime residual fusion、**soft regime ensemble（软门控集成）**——早期一度为最强候选，但在 rolling 上波动较大、未形成可泛化优势。
4. **残差建模与多专家**：common residual、residual-level stacking、多尺度专家（short/mid/long/full）、原始序列专家（seq_short/mid/long）（V2 / V3.1 / V3.3 B0–B7 系统对比）——复杂结构在小样本 rolling 上不稳定，最稳健者反而是朴素 ridge backbone。
5. **传统选模型决赛（expanding）**：`M_lgbm_d4`（上限型，全面领先）/ `M_tree_d8`（**ExtraTrees 在 expanding 上性能显著退化、出现负折，予以否决**，因随机森林仅对全历史做平均、无近因加权）/ `M_tree_d7`。
6. **样本外检查**：X1（11 月 holdout 0.0737）与 lgbm_d4（11 月 holdout 0.0897）各自通过零重合样本外检查。

**B. 深度学习阶段（5/31–6/4）**

1. **TCN-on-raw（否决）**：时序卷积直接以 raw 为输入，但张量契约 `[B,L,C]` 单次仅处理一只股票、无截面归一化 → **无法感知 cross-z / cross-rank 这一维截面 alpha**；9 点验证 mean=0.0091、R² 全负、种子方差大。**根因为截面盲区**（非训练轮数不足），直接促成向截面建模的转向。
2. **GRU-on-433（基线，0.0585）**：对个股日内路径编码，结构成立但 R² 微负——暴露出 **MSE 与 corr 的矛盾**。
3. **XSECTION-on-433（0.0624）**：加入 set-attention 截面支路，但仍以 433 人工摘要为输入，且 146 天时间截面过少 → 增量极小（best_epoch 1–2）。这使我们认清**瓶颈不在模型容量，而在于「输入的是已被传统模型充分提取的人工摘要」**。
4. **XSECTION-on-raw（突破）**：使截面模型**直接以 raw 59 通道为输入**。该支路与传统**真实去相关**，融合产生净增益——印证「12% 的真正入口 = 直接建模 raw 盘口」。

**C. 两条「试过但未作主线」的深度学习变体（负结果消融，分支保留）**

5. **DeepLOB 式 raw 前端（分支 `feat/xsection-raw-deeplob`）**：在单股票时序支路最前端加入轻量 1D 卷积前端（`raw → Conv → GRU → 截面层`），
   保留 DeepLOB「以卷积前端从原始盘口提取局部模式」的核心思想（未完全复刻原论文卷积塔）。实测 cert 0.0763 / delivery 0.0770，
   **反不及 baseline XSECTION_RAW（cert 0.0860 / delivery 0.0792）** → 在本数据上卷积前端未带来增益。此为论文启发式尝试，记为负结果。
6. **残差训练（分支 `feat/xsection-raw-residual`）**：将「去相关」直接做进训练目标——深度学习学习残差 `r = y − pred_trad`，
   输出 `pred_final = pred_trad + pred_dl_res`（实现见 `src/trad_residuals.py` 与 `src/dl_trainer.py` 的 `target_mode=residual_trad`）。
   实测组合 cert 0.0760 / delivery 0.0813，**不及等权融合主线（其 cert 三折约 0.10）**。
   根因（诚实）：现成的 OOS 传统预测仅覆盖训练尾段 → 最近一折的有效残差训练历史被压缩过短、对 cert 判官不利；并非残差思想本身有误。

### 4.2 最终传统支路设计（X1 ridge + M_lgbm_d4 等权融合）

锁定**等权 raw_mean 融合**的核心理由为 **minimax 正解**：在「权重几乎不影响 pooled corr」的平台区内，等权点的**最坏折表现最优**（0.0491 优于两成员单独）；
隐藏集分布未知，故采用 minimax 准则——等权是「高均值区内最能抵抗最坏情形」的可辩护默认（不宣称为「最优」）。融合模式选用 raw / zscore（rank 明显劣于二者，因老师采用 Pearson 而非 Spearman）。

### 4.3 最终深度学习支路设计（`CrossSectionCartridge`，`models/dl_models.py`）

因子化双支路结构：
- **共享 GRU 时序支路**：对每只股票的日内序列做路径编码；
- **跨股票 set-attention 截面支路**：在同一 `(date, interval)` 截面内对所有股票做 `MultiheadAttention`，**置换等变、无位置编码、无 ID 嵌入**（截面内个股无固有顺序），并带 padding mask；
- **零初门控残差** `z = h + γ·Δ`（γ 初值为 0，初始退化为纯 GRU，训练中自适应开启截面交互）；
- **直接以 raw 为输入**：`RawChannelAdapter` 对 59 个原始微结构通道做最小语义归一化后作为通道输入，盘口形状与订单流不再被压缩为静态比率。

### 4.4 训练上的独特设计

- **`MSE + λ·(1 − 截面内 Pearson)` 损失**（`src/dl_losses.py`）：corr 项为同一截面内跨个股的 Pearson（即老师评分所用的量），使「直接优化 corr」对齐到被评分的目标（λ=0.3）。
- **训练段全局 OLS rescale（`a·ŷ+b`，fit-on-train / apply-on-val，零泄漏）**：重标定后 R²=corr²，corr>0 即自动 R²≥0 ——**化解 MSE 与 corr 的矛盾**，使 λ 的选择仅依最大化 val corr，三指标不再相互牺牲。
- **多 seed 集成（降低单 seed 方差的低成本手段）**：离线判决采用 **2 个 seed（42、43）平均**，三个选型折融合即全部突破 0.10（见 §7.1，0.1040 / 0.1016 / 0.1098）；**serve 交付链进一步配置 K=3 seed（42 / 43 / 44）**，§7.2 的 0.0919 即 3-seed 平均融合的端到端结果。
- **去相关融合**：深度学习不与传统争夺同一份信息（raw 对人工摘要），ρ≈0.45–0.55 中等相关 → 集成在数学上具备可实现的增益空间。

### 4.5 参考文献（较新方法引用）

- **DeepLOB**（Zhang, Zohren, Roberts, 2019）：以深度网络直接建模原始限价订单簿，启发「直接建模 raw 盘口」主线与 §4.1-C5 的卷积前端变体。
- **Set Transformer / Deep Sets**（Lee et al. 2019；Zaheer et al. 2017）：置换等变的集合建模，启发截面支路的 set-attention 设计（截面内个股为无序集合）。
- **STGNN / 关系排序**思想：本项目以因子化双支路（GRU × set-attention）替代重型时空图，在样本量与表达力之间做了权衡。

## 5. 大模型创新性调用（创新点）

本项目**全程由大模型（AI Agent）深度驱动研发**，对应老师评分维度「大模型创新性调用（如 Agent 自动生成因子挖掘）」：

- **因子挖掘假设生成**：借助大模型从市场微观结构原理（动量 / 反转、订单流不平衡、成交冲击、条件动量）反向推导候选特征族，
  再落实到 `feature_registry.py` 的 stage 实现——特征并非随机组合，而是「原理→假设→可证伪实验」的闭环。
- **实验编排与诊断归因**：大模型负责设计「宽进严出」评测漏斗，并对每次 run 的 leaderboard 做根因归因
  （例：P1 / P2 判负的统一根因为基线已含该族截面归一主体；TCN 失败根因为截面盲区），将失败转化为下一步的研究方向。
- **严谨性把关**：大模型主动识别并封堵泄漏 / 过拟合风险（walk-forward purge / embargo、训练段统计、交付折只读不回灌），
  并发现多项交付隐患（入口日期写死会导致其他时段数据崩溃、ridge 内存依赖型 NaN、提交件旧快照缺失健壮性）并逐一修复。
- **代码实现与重构**：脊柱 + 可换卡带架构、瘦内存管线、防御式降级等工程实现均由大模型完成。

## 6. 评测协议与严谨性（防过拟合）

老师以**完全零重合的数据**重训重评 → 考查的是**方法的跨时间泛化能力**，因此离线评测严格模拟该场景：

- **锚定扩展 walk-forward + purge / embargo**：训练锚定 6/1 并向后扩展；选型采用 11 月末倒贴的 3 个 20 交易日折，外加 1 个 Dec 交付对齐折；折间设 1 日 embargo（`fret12` 日内不跨夜，已等价于 purge）。
- **以最坏折读数为主裁**：不取单折最优成绩，而看跨折中最坏折是否仍为正（GO 判据）。
- **零泄漏**：归一化仅用训练区统计；OLS rescale 仅在训练段拟合；深度学习 `fit` 与 `predict` 分离，predict 复用训练期 Normalizer。
- **防时段过拟合三层措施**：重正则（dropout 0.2 / weight_decay 1e-3 / 早停 patience 5）+ 多 seed 降方差 + 传统保底回落。
- **交付折只读不回灌**：冠军超参定死后才运行 Dec 评测，杜绝据 Dec 调参。
- **入口健壮性**：`meow.py` 自动适配数据实际所在时间段（`Calendar.mergeDataDirDays` + `__main__` 日期回退），老师更换数据路径而不修改入口日期亦不会崩溃。

## 7. 结果分析与总结

### 7.0 核心结果速览（关键结论）

> **① 最终交付（老师 `python meow.py` 端到端实跑、3-seed 融合）：Pearson = 0.0919（突破 0.09），R² 与 MSE 同步改善；在无 torch / GPU 的环境下自动回落至传统保底 0.0812，保证不崩溃。**

### 7.3 总结

- 在不增加数据种类的前提下，依靠「**直接建模 raw 盘口的去相关深度学习支路 + 传统多因子保底**」的融合，将交付分从传统 0.0812

  老师零重合数据上的具体数值会有不同，但方法已经四窗 walk-forward 验证、具备泛化依据。

## 8. 创新点说明（汇总）

1. **直接建模 raw 盘口的去相关深度学习支路**：截面序列模型（GRU 时序支路 + set-attention 截面支路 + 零初门控残差）直接以 raw 59 通道为输入，
   与传统多因子**真实去相关（ρ≈0.45–0.55）**，是融合净增益的来源（稀有性：多数队伍在人工特征上建模，少有直接建模 raw 截面者）。
2. **去相关融合的 minimax 等权**：以「最坏折表现最优」论证零自由参数等权，而非据单窗调权（防过拟合）。
3. **`MSE+λ·corr` 损失 + 训练段 OLS rescale**：化解「MSE 与 corr 矛盾」——使 R²=corr² 自动 ≥0，超参选择仅看 corr，三指标不相互牺牲。
4. **防御式交付**：深度学习支路任一环节出错即自动回落纯传统，保证任意环境不崩溃、不产生 NaN，保底分无条件可达。
5. **大模型驱动研发**（见 §5）：因子假设生成 / 实验编排 / 诊断归因 / 严谨性把关全程为 AI Agent 闭环。
